Gradient Descent: BGD
BGD computes the gradient based on the average of the gradients of all data samples in the training set.
1. Therefore, for each iteration, the gradient has to be calculated for the entire dataset, making BGD a computationally expensive process for huge datasets.
2. However, it also gives BGD the advantage of being more stable and avoiding overfitting compared to SGD.
3. The learning parameter has to be chosen carefully, so that with each iteration the step size doesn't get bigger or diverge from the minima.
4. BGD is usually used in simpler models with lesser data. 5. It is widely used in linear regression and logistic regression, not so much in deep learning
where the training set is typically large and the models have many parameters.

Gradient Descent: SGD
SGD computes the gradient for only one random sample at each iteration.
1. This property of SGD helps in it being faster and efficient as it does not have to process all the data in each of its iterations.
2. However, the randomness of SGD contributes to the fact that it can in some cases give the sub-optimal solutions/local minima as the result rather than the global minimum.
3. One of the techniques to overcome this fault is to decrease the learning rate of the model over time, which helps in reducing the updates in the parameter with each iteration.
4. SGD also has its variants, like Mini-Batch SGD, where the Gradient descent is done for a random subset of data, and Momentum SGD, where a term is added to the gradient update to help with the optimization and avoiding getting stuck at a local minima.
5. SGD is majorly used in Deep Learning and has found applications in classification, regression, and neural machine translation.

Gradient Descent: MBGD
MBGD is a combination of SGD and BGD.
1. MBGD, in comparison to BGD, computed the gradient over a subset of the data, called the mini-batch.
2. This helps in computing faster than BGD and avoiding overfitting as compared to SGD. 3. The mini-batch size is a trade-off between speed and stability, with smaller sizes leading to
faster convergence but increased variability in the optimization, and larger sizes leading to more stable convergence but slower processing times. 4. The learning rate, which determines the size of the parameter update at each iteration, must be carefully tuned to ensure that the optimization converges to the minimum and does not oscillate or diverge. 5. Mini-Batch Gradient Descent is widely used in deep learning and has been applied to a variety of tasks, including classification, regression, and neural machine translation

Backpropagation and Chain Rule Backpropagation is an algorithm that uses the chain rule of calculus to efficiently calculate the gradient of the loss function in a neural network. This gradient reveals how each weight and bias in the network affects the overall error, enabling the model to learn and adjust its parameters through optimization algorithms like gradient descent
Generally speaking, training neural networks with backpropagation entails the following steps:
1. A forward pass, making predictions on training data. 2. A loss function measures the error of the model's predictions during that forward pass. 3. Backpropagation of error, or a backward pass, to calculate the partial derivatives of the
loss function. 4. Gradient descent, to update model weights.

Backpropagation and Chain Rule

Backpropagation is the cornerstone of training neural networks. It involves the iterative process of computing gradients and updating the network's parameters to minimize a chosen loss function. At the core of backpropagation is the application of the chain rule to compute these gradients efficiently.

1. Forward Pass: The input data is passed through the network layer by layer. Each layer performs a weighted sum and applies an activation function to produce the output. These outputs are stored for later use in the backward pass. 2. Backward Pass (Backpropagation): Starting from the output layer, the gradient of the loss function with respect to the output is computed. The chain rule is applied iteratively to compute the gradient of the loss function with respect to each layer's parameters and inputs. Gradients are propagated backward through the network.

ERM: Empirical Risk Minimization
Key Aspects of ERM in Deep Learning:
· Goal: The primary objective is to select a model (hypothesis) that has the lowest possible error on the training data, assuming this will lead to good performance on new data.
· Loss Function: A critical component that quantifies the difference between predictions and actual targets (e.g., Mean Squared Error, Cross-Entropy).
· Optimization: Since finding the absolute minimum is difficult, iterative methods like Stochastic Gradient Descent (SGD) are typically used to minimize the empirical risk.
· Limitations: Pure ERM often leads to overfitting, where the model learns the training data too well (including noise) and performs poorly on new data.
· Regularization: To counter overfitting, regularization techniques (like L1/L2 regularization) are often added to ERM to constrain model complexity
