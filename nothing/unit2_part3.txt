Logistic Regression Logistic regression is a common machine learning algorithm for binary classification and predicts a binary categorical variable through a logistic function.
The logistic regression model passes the outcome of a linear function of features through a logistic function to calculate the probability of an occurrence. The model then maps the probability to binary outcomes.

The sigmoid function maps any real-valued number x  R to a value between 0 and 1. This ensures that output falls within a valid probability range.
Example applications: 1. Email spam detection (spam / not spam) 2. Disease prediction (disease / no disease) 3. Student pass/fail prediction

Underfitting, Overfitting and Good fitting

1. Underfitting : Straight line trying to fit a curved dataset but cannot capture the data's patterns, leading to poor performance on both training and test sets.
2. Overfitting: A squiggly curve passing through all training points, failing to generalize performing well on training data but poorly on test data.
3. Appropriate Fitting: Curve that follows the data trend without overcomplicating to capture the true patterns in the data.

Overfitting
Various approaches have been proposed and applied in practice to avoid overfitting. They target different aspects of a machine learning task.
1. Design of datasets: In general, the model generalizes better with more training data. In addition to gathering more data, data augmentation is also widely adopted.
2. Use of a simpler model: As a larger function space is more prone to overfitting, a simpler model is usually preferred.
3. Regularization in various forms: Explicit regularization includes adding a penalty term, dropout for Deep Neural Networks (DNN), weight decay, etc. Implicit regularizations include early stopping and batch normalization, etc.

Concept:

Early Stopping

Early stopping is a regularization technique where we stop training when performance on the

validation set stops improving.

During training: ñ Training loss usually keeps decreasing ñ Validation loss decreases first, then may start increasing (overfitting starts)

Early stopping stops training at the point of best validation performance, not when training loss is minimum.
Why It Works:

ñ Prevents the model from memorizing training data ñ Keeps the model at a good biasÙvariance tradeoff ñ Acts like implicit regularization (limits effective model complexity)

Dropout
Dropout is a regularization technique where, during training, we randomly turn off (drop) a fraction of neurons in each layer.
ñ If dropout rate = 0.5: ñ 50% of neurons are randomly removed in each iteration
ñ At test time: ñ All neurons are used, but outputs are scaled appropriately
Why It Works
ñ Prevents co-adaptation of neurons (neurons depending too much on each other) ñ Forces the network to learn robust, redundant features ñ Acts like training many different smaller networks and averaging them

Batch Normalization normalizes the activations of a layer for each mini-batch so that they have: ñ Mean  0 ñ Variance  1 Then it applies a learnable scale and shift.

Greedy layer wise training
Greedy layer-wise training is a training strategy for deep neural networks where, we train the network one layer at a time, starting from the first layer, instead of training all layers together end-to-end.
1. Each layer is trained to learn useful representations of the input. 2. Once a layer is trained, its parameters are frozen. 3. Then the next layer is added and trained on top of the previous one. 4. Finally, the whole network may be fine-tuned using backpropagation. 5. It is called "greedy" because each layer is optimized locally without considering the global
optimum of the full network.

Greedy layer wise training
Why Was It Needed? (Historical Motivation) In early deep learning (pre-2012 era), training very deep networks with backpropagation was difficult because of:
1. Vanishing gradients 2. Poor initialization 3. Getting stuck in bad local minima Networks failed to converge or learned very poor solutions. Greedy layer-wise training was introduced to: ñ Provide good initialization for deep networks ñ Make training stable and feasible ñ Learn features progressively from low-level to high-level
